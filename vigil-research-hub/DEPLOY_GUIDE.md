# Vigil Research Hub — Ръководство за пускане на сайта

Този проект е готов за деплой. Следвай стъпките по ред — отнема около 15–20 минути, без да пишеш нито ред код.

Резултат: реален сайт с адрес (напр. `vigil-research-hub.vercel.app`), с регистрация по имейл и парола, и лично пространство за всеки потребител.

---

## Стъпка 1 — Качи кода в GitHub (5 мин)

1. Отиди на [github.com](https://github.com) и направи безплатен акаунт, ако нямаш.
2. Натисни зеленото копче **New repository**.
3. Дай му име: `vigil-research-hub`. Остави го **Public** или **Private** — без значение.
4. НЕ маркирай "Add a README file". Натисни **Create repository**.
5. На следващия екран ще видиш инструкции "...or push an existing repository from the command line". Игнорирай ги за момента — ще качим файловете директно през браузъра:
   - Натисни **uploading an existing file** (линк в средата на страницата).
   - Провлачи цялата папка `vigil-research-hub` (всички файлове от този проект) в браузъра.
   - Натисни **Commit changes**.

---

## Стъпка 2 — Направи безплатен Supabase проект (5 мин)

Supabase е безплатната база данни, която управлява акаунтите (регистрация, вход) и пази запазените статии на всеки потребител.

1. Отиди на [supabase.com](https://supabase.com) и се регистрирай безплатно (може с GitHub акаунта от стъпка 1).
2. Натисни **New Project**.
   - Name: `vigil-research-hub`
   - Database Password: генерирай си силна парола и я запази някъде — няма да ти трябва често, но е добре да я имаш.
   - Region: избери най-близкия до България (обикновено `eu-central-1` — Frankfurt).
3. Изчакай ~2 минути, докато Supabase създаде проекта.

### Създай таблицата за запазени статии

4. В лявото меню натисни **SQL Editor**.
5. Натисни **New query**.
6. Отвори файла `supabase_schema.sql` от този проект, копирай **целия** му текст, и го постави в редактора.
7. Натисни **Run** (или Ctrl/Cmd + Enter). Трябва да видиш "Success. No rows returned."

Това създаде таблицата `saved_articles` и настрои сигурността така, че всеки потребител вижда **само своите** запазени статии.

### Вземи ключовете за връзка

8. В лявото меню натисни **Project Settings** (иконата зъбно колело) → **API**.
9. Ще видиш две неща, които ти трябват:
   - **Project URL** (изглежда като `https://xxxxx.supabase.co`)
   - **anon public** ключ (дълъг текст под "Project API keys")
10. Запази ги някъде временно — ще ги ползваш в следващата стъпка.

### Настрой имейл потвържденията (по избор, но препоръчително)

11. В лявото меню: **Authentication** → **Providers** → **Email**.
12. По подразбиране Supabase изисква потвърждение по имейл при регистрация. Ако искаш хората да могат веднага да влизат без да чакат имейл (по-просто за тестване), изключи "Confirm email" тук. За истински публичен сайт по-добре го остави включено.

## Стъпка 3 — Деплой във Vercel (5 мин)

Vercel е безплатният хостинг, който прави сайта достъпен с реален адрес.

1. Отиди на [vercel.com](https://vercel.com) и се регистрирай безплатно с твоя GitHub акаунт.
2. Натисни **Add New** → **Project**.
3. Ще видиш списък с твоите GitHub хранилища — намери `vigil-research-hub` и натисни **Import**.
4. Vercel автоматично разпознава, че е Vite проект. Не променяй нищо в Build настройките.
5. Преди да натиснеш Deploy, отвори секцията **Environment Variables** и добави:

   | Name | Value |
   |---|---|
   | `VITE_SUPABASE_URL` | Project URL от Supabase (стъпка 2.9) |
   | `VITE_SUPABASE_ANON_KEY` | anon public ключът от Supabase (стъпка 2.9) |

6. Натисни **Deploy**.
7. Изчакай 1–2 минути. Vercel ще ти даде линк като `vigil-research-hub.vercel.app` — това е твоят реален сайт.

---

## Стъпка 4 — Тествай

1. Отвори линка от Vercel на телефона си.
2. Натисни **Вход** → **Регистрирай се** → въведи имейл и парола.
3. Ако си оставил email confirmation включен — провери имейла си и натисни линка за потвърждение.
4. Влез. Отиди на **Търсене**, потърси нещо, натисни иконата за запазване.
5. Отиди на **Запазени** — трябва да видиш статията там.
6. Отвори сайта от друг браузър или инкогнито прозорец, регистрирай втори тестов акаунт — увери се, че вторият акаунт **не вижда** статиите на първия. Това потвърждава, че личните пространства работят правилно.

## По-нататък — добавяне на собствен домейн (по избор)

Ако искаш адрес като `vigil-research.com` вместо `vigil-research-hub.vercel.app`:

1. Купи домейн от Namecheap, Google Domains, или подобен регистратор (~€10–15/година).
2. Във Vercel: Project → Settings → Domains → добави своя домейн.
3. Vercel ще ти покаже DNS записи, които да добавиш при регистратора на домейна.
4. След 10–60 минути домейнът ще сочи към сайта ти.

## Какво прави всеки файл (за твоя справка)

- `src/sources.js` — списъкът с 12-те източника (PubMed, FDA, и т.н.). Тук добавяш нови източници в бъдеще.
- `src/pages/SearchPage.jsx` — страницата за търсене с филтри.
- `src/pages/SavedPage.jsx` — личното пространство със запазени статии.
- `src/pages/AuthPage.jsx` — вход и регистрация.
- `src/AuthContext.jsx` — управлява кой е влязъл в момента навсякъде в сайта.
- `supabase_schema.sql` — структурата на базата данни и правилата за сигурност.

## Добавяне на нов източник по-късно

Отвори `src/sources.js` в GitHub (натисни на файла → иконата молив за редакция) и добави нов обект в масива `SOURCES`, следвайки същия формат. Commit-ни промяната — Vercel автоматично ще обнови сайта за няколко секунди.
